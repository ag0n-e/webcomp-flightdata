The components can be used multiple times with different configurations and use a minimum height of 200px. If the surrounding container has a height defined, the component uses 100% of the available height. If the provided height is not sufficient, a scrollbar will be displayed. The necessary height results from the elements to be displayed + possibly defined spacing (margin/padding). The tableheight attribute can be used to limit single tables to a fixed height. The surrounding container must have a sufficient height defined so that displayed components have enough space without triggering container scrollbar. Internal spacing and styles within the shadow root can be overridden by providing an own custom CSS file (styleurl=""). The default style of the component is designed for a reasonable simultaneous display of all offered features. Due to the various possible use cases and different usage scenarios when combining individual features, the use of custom styles is recommended in order to better handle any borderline cases.
Currently relevant airport is indicated by "filters.airport". Only flights that directly affect this airport are displayed. Flights can be additionally filtered by nation. See src/mappings/airport_types.js for current possible values. Using the airports attribute, individual destinations can be included or excluded. If an exclusion is present, existing inclusions will be ignored. Avoid mutually exclusive filter-combinations. If columns attribute is present, missing column configurations are assumed to be false. Translations can be changed or added via the messages attribute.
